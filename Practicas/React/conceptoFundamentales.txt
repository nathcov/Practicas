1.- Componentes
Son los bloques de construcción de la aplicación. Son piezas de código independientes y
reutilizables que representan una parte de la interfaz de usuario. Pueden ser
funcionales (funciones de JavaScript) o de clase (clases de ES6).

// Ejemplo de un componente funcional básico
function Saludo() {
    return <h1>Hola, mundo!</h1>; //Retorna el elemento de React
}

2.- JSX (JavaScript XML)
Es una extensión de la sintaxis de JavaScript que permite escribir una estructura similar a
HTML dentro de los archivos de JavaScript. Facilita la visualización de la estructura de
los componentes.

//Ejemplo de JSX asignado a una constante
const elemento=<h1>Este es un título principal</h1>;

3.- Props (Propiedades)
Son argumentos que se pasan a un componente para personalizarlo. Los props son unidireccionales:
Siempre viajan del componente padre al componente hijo.

//El componente hijo recibe 'props' como argumento
function Testimonio(props) {
    return (
        <div className='contenedor-testimonio'>
            <p>Nombre: {props.nombre}</p>
            <p>Cargo: {props.cargo}</p>
        </div>
    )
}

//El componente padre pasa los valores al crear el componente
<Testimonio
    nombre='Emma Bostian'
    cargo='Ingenieria de Software'
/>

4.- Estado (State)
Representa los valores actuales de las propiedades de un componente en un momento
específico. A diferencia de los props, el estado es interno y permite que la aplicación sea
dinámica e interactiva.

// En el proyecto del contador, el estado es el número de clics
const [numclics, setNumClics] = useState(0); // 0 es el valor inicial

5.- Hooks
Son funciones especiales que permiten a los componentes funcionales trabajar con estados y
otros aspectos de React sin necesidad de escribir una clase. El más común es useState,
que devuelve el valor del estado y una función para actualizarlo.

// Implementacion de useState para actualizar el contador
const manejarClic=() => {
    setnumclics(numclics+1); // La funcion set actualiza el estado y vuelve a renderizas
};

6.- Event Listeners (Eventos)
Son funciones que se ejecutan cuando ocurre una accion especifica del usuario, como un clic en un boton.
En React, se asignan directamente en el JSX usando atributos como onClick

// Asignacion de un evento de clic a un boton
<button onClick={manejarClic}>
Haz clic aqui
</button>

Estos conceptos permiten crear aplicaciones donde React solo actualiza las partes de la interfaz 
que han cambiado, optimizando el desempeño.
